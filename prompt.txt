buatkan issue.md, untuk membuat planing. Nanti dokumen planingnya akan digunakan programmer atau model yang lebih murah

isi dari planing nya adalah sebagai berikut:

buatkan api untuk logout
endpoint : DELETE /api/users/logout

headers :
- Authorization : Bearer <token> (token ada token yang ada di table users)

response body (success): 
{
    "status": 200,
    "message": "Logout success",
    "data": null
}

Jika sukses logout, maka token yang ada di table users harus dihapus

response body (error):
{
    "status": 401,
    "message": "Logout failed",
    "data": null
}

Struktur folder didalam src
- route : ini berisi routing ElysiaJS
- service : berisi logic bisnis aplikasi

Struktur penamaan file
- route : nama file harus diawali dengan "route-" contoh : route-user.ts
- service : nama file harus diawali dengan "service-" contoh : service-user.ts



Jelaskan step tahapan yang harus dilakukan untuk mengimplementasikan fitur ini. anggap nanti yang mengimplementasikan adalah junior developer atau ai model yang lebih murah, jadi jelaskan sedetail mungkin.
